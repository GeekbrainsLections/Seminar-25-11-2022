# Инструкция по работе с Git

## Что такое Git?
Git - это наиболее популярная реализация распредеённой системы контроля версий. Наиболее полпулярной реализацией 

## Подготовка репозитория
Для создания репозитория в папке предпологаемого репозитория необходимо написать команду *git init*. Для этого в терминале с открытой папкой-репозиторем пишем *git init*

## Создание коммитов

### Добавление файлов к коммиту
Для добавления файлов к коммиту используется команда *git add*. Для того, чтобы добавить файл к новому коммиту необходимо в терминале с открытой папкой-репозиторием написать *git add <имя файла>*.

### Создание коммитов
Для создание коммитов используется команда *git commit*. Для этого в терминале с папкой-репозиторием необходимо написать команду *git commit -m "<сообщение к коммиту>"*. Сообщение к коммиту писать ***ОБЯЗАТЕЛЬНО***. 

## Журнал изменений
Для просмотра истории изменений используется команда *git log*. Для этого в терминале с папкой-репозиторем необходимо набрать *git log*.

## Перемещение между коммитами
Для перемещения между коммитами используется команда *git checkout*. Для этого в терминале с папкой-репозиторием необходимо написать *git checkout <номер коммита>*. Номера коммитов можно посмотреть в истории коммитов, описанной в предыдущем пунткте.

## Ветки в Git

### Просмотр списка веток
Для просмотра списка веток используется команда *git branch*. Для этого в терминале с папкой-репозиторием необходимо написать команду *git branch*

### Переход между ветками
Для перехода между ветками используется команда *git checkout*. Для этого в терминале с папкой-репозиторием необходимо написать *git checkout <название ветки>*.


### Создание новой ветки
Для создания новой ветки используется команда *git branch*. Для этого в терминале с папкой-репозиторием необходимо написать *git branch <название ветки>*.

## Слияние веток и разрешение конфликтов

## Удаление веток

Удаление веток можно призводить путем прописывания команды 'git branch -d <название ветки, которую нужно удалить>' 
  
## Удаленный репозиторий  

До сих пор мы использовали репозиторий только локально. 
Это удобно, пока мы работаем над проектом самостоятельно оффлайн и нет необходимости предоставлять доступ к проекту кому-нибудь ещё. Это конечно можно реализовать передавая заархивированную папку с репозиторием на флешке и пересылая её электронной почтой. Помним что локальный репозиторий хранится на нашей машине, в рабочей папке проекта. Это та самая скрытая папка .git. Но при этом проект находится только на нашей машине и в случае поломки железа или случайной потери данных мы не сможем восстановить проект.  
Функцию сохранности данных а также функционал командной работы над проектом можно реализовать с помощью удаленного репозитория.  
**Удаленный репозиторий** это репозиторий, который хранится в облаке, на сторонних сервисах, специально созданных под работу с проектами git.  
Плюсы удаленного репозитория:
* выполняет роль резервной копии  
* возможность работать в команде  
* некоторые дополнительные возможности, которые предоставляет хостинг. Например, визуализация истории или возможность работать над проектом прямо в веб-интерфейсе.  
  
## Выбор хостинга Git  
Существует целый ряд сервисов предоставляющих хостинг для Git-репозиториев. Самыми известными являюются:
* [GitHub](https://github.com)
* [GitLab](https://gitlab.com)
* [SourceForge](https://sourceforge.net)
* [BitBucket](https://bitbucket.org)  
  
У каждого хостинга есть свои плюсы и минусы но мы выберем GitHub как самый популярный.  

## Клонирование репозитория  
  
GitHub позволяет скачивать публичные репозитории на локальный компьютер без создания учетной записи. Для этого нужно на локальном компьютере в терминале Git зайти в папку, в которой мы хотим разместить скачанный репозиторий и прописать команду:  

    git clone <ссылка на нужный репозиторий на GitHub>  

При этом репозиторий скачается в выбранную папку в виде вложенной папки с тем же именем, c с которым он представлен на GitHub. Если мы хотим изменить имя папки, нужно в конце команды, после ссылки на репозиторий через пробел указать желаемое имя:  

    git clone <ссылка на нужный репозиторий на GitHub> <новое имя папки>  

 Чтобы зайти в клонированный репозиторий, нужно сначала перейти в его папку командой:  
  
    cd /<Имя папки>  

## Собственный удаленный репозиторий  

Для своих проектов нам понадобится собственный репозиторий. Для этого сначала нужно зарегистрироваться на [GitHub](https://github.com/signup).  
После прохождения регистрации и подтверждения контактных данных.  
После регистрации создание репозитория доступно с главной страницы github. При создании нужно указать название проекта и тип (публичный или приватный). На остальное пока не обращаем внимания.  
  
Есть два способа создания проекта:  

* Первый, когда мы начинаем новый проект. Удобнее будет создать репозиторий на github и склонировать пустой проект на локальную машину.
* Второй, когда у нас уже есть проект. Нужно зайти в папку проекта и связать его с уже существующим репозиторием на github.  
  
Допустим, у нас на локальной машине уже есть проект MyProject:
* Создаем в GitHub репозиторий MyProject и копируем ссылку на него. 
* Заходим в локальную папку проекта и выполняем команды:
  
      git remote add origin <ссылка на наш репозиторий>    #Создаём связь с удаленным репозиторием
      git branch -M master    #Указываем какая ветка будет основная
      git push -u origin master    #Отправляем файлы в удаленный репозиторий.  

## Обмен данными с удаленным репозиторием  
Для отправки данных (коммитов и веток) на сервер (в удаленный репозиторий) используется команда **git push**.  

Зачем пушить на сервер?  

1. для работы в команде, чтобы делиться своим кодом с коллегами
2. чтобы иметь резервную копию на случай потери данных на своей машине  

Для отправки данных на сервер пишем терминале:  

    git push origin master  

Проверить есть ли у нас незапушенные коммиты можно командой **git status**:  

    git status
    On branch master
    Your branch is ahead of 'origin/master' by 5 commits.
      (use "git push" to publish your local commits)
    (use "git push" to publish your local commits)  

В приведенном примере ключевая фраза "Your branch is ahead of 'origin/master' by 5 commits." указывает на то, что у нас есть 5 коммитов не отправленных на сервер.  
  
  Для загрузки данных с сервера используется команда **git pull**. Похоже на клонирование репозитория, но с той разницей, что скачиваются не все коммиты, а только новые.  
  Пулиться с сервера нужно чтобы получать изменения от ваших коллег. Или от себя самого, если работаете на разных машинах.  
  Запускается команда следующим образом:  

      git pull origin master  

## Включение написанного вами кода в чужой репозиторий  

Мы можем в полном объёме работать со своим удаленным репозиторием, а так же в репозиториях, где мы включены в команду разработчиков. Когда же мы хотим поучаствовать в разработке некого продукта и можем предложить некоторые свои наработки используется механизм **pull request**.
Основная идея механизма в том, что мы имеем возможность скачать чужой репозиторий, поработать с ним и отправить коммиты с нашими наработками хозяину репозитория с просьбой ознакомится с нашей работой и слить её с оригинальным репозиторием, если хозяина всё устроит. Такую операцию можно произвести имея свой аккаунт на том-же ресурсе, где размещается основной проект.
Для этого:  
1. Находим на странице с репозиторием интересующего нас проекта кнопку **"Fork"** и нажав её создаём точную копию проекта в своём удаленном репозитории.
2. Клонируем **СВОЙ** удаленный репозиторий с проектом на локальный компьютер.
3. Создаём ветку, в которой будем предлагать свои изменения.
4. Вносим свои изменения только в эту ветку и коммитим её.
5. Пушим новую ветку на сервер
6. После проделанных операций на странице с нашей версией проекта появится кнопка **"Pull request"**
